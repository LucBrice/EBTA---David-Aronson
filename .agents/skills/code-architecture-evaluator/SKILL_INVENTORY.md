## 🎯 WORKFLOW PHASES

### Phase 1 : INGESTION (Silent)
```
User provides codebase (repomix, files, URL)
    ↓
Skill reads and maps :
  • Architecture & patterns
  • Dependencies & versions
  • Data flows & relationships
  • Test coverage
  • Tech debt areas
    ↓
NO verbal summary (silent processing)
```

### Phase 2 : PLAN EXTRACTION
```
User describes refactoring plan
    ↓
Skill extracts :
  • Modules affected
  • New dependencies
  • Architecture changes
  • Implicit assumptions
    ↓
Ready for audit
```

### Phase 3 : CRITICAL AUDIT (6 Sections)
```
1️⃣ Executive Summary (verdict + risk level)
2️⃣ Strengths (3-5 positive alignments)
3️⃣ Weaknesses (3-7 issues identified)
4️⃣ Blind Spots (8-point checklist)
   - Data migration
   - API contracts
   - Test coverage
   - Progressive deployment
   - Transition phase
   - Monitoring
   - Inverted dependencies
   - Documentation
5️⃣ Standards (SOLID violations)
6️⃣ Rectified Plan (phased, with risks & metrics)
```
